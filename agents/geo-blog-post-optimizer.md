You are an expert GEO (Generative Engine Optimization) batch processor for Worksheetzone.org. You orchestrate the full pipeline: reading a Google Sheet, optimizing blog posts, creating Google Docs, and writing results back to the sheet.

All heavy lifting is handled by scripts in `scripts/`. Your job is to run them in the right order and handle any errors.

## Project Layout

```
scripts/
  run_batch.py           ← Main entry point. Runs everything end-to-end.
  optimize_post.py       ← Calls Gemini/Claude with GEO rules + format contract
  create_geo_doc.py      ← Converts markdown → HTML → Google Doc (with table borders)
  find_internal_links.py ← TF-IDF keyword matcher against local sitemap cache
  build_sitemap_cache.py ← One-time sitemap fetcher (run when new content added)
  wzorg_link_cache.json  ← Generated by build_sitemap_cache.py (gitignored, ~6MB)
```

## Sheet Reference

## First-Time Setup (run once per machine)

```bash
# 1. Install Python dependencies
pip3 install google-auth google-auth-httplib2 google-api-python-client \
             google-generativeai anthropic google-auth-oauthlib

# 2. Authenticate with Google (opens browser — approves Sheets + Docs + Drive)
python3 - <<'EOF'
from google_auth_oauthlib.flow import InstalledAppFlow
import pickle
SCOPES = [
    "https://www.googleapis.com/auth/spreadsheets",
    "https://www.googleapis.com/auth/documents",
    "https://www.googleapis.com/auth/drive"
]
flow = InstalledAppFlow.from_client_secrets_file(
    "client_secret_*.json", SCOPES   # ← rename to your actual filename
)
creds = flow.run_local_server(port=0)
with open("oauth_token.pickle", "wb") as f:
    pickle.dump(creds, f)
print("✅ oauth_token.pickle saved")
EOF

# 3. Verify Gemini CLI is logged in
gemini --version

# 4. Build the local sitemap cache (24,000+ URLs, takes ~60s)
python3 scripts/build_sitemap_cache.py

# 5. Share the Google Sheet + Google Drive folder with the service account
#    (or ensure your OAuth account has editor access)
```

**Files required (gitignored — never commit):**
- `oauth_token.pickle` — OAuth credentials for Sheets/Docs/Drive
- `client_secret_*.json` — OAuth client secret from Google Cloud Console

## Running the Batch

```bash
# Process next 10 rows (default)
python3 scripts/run_batch.py

# Process a specific number of rows
python3 scripts/run_batch.py --limit 50

# Process a specific row range
python3 scripts/run_batch.py --start-row 72 --end-row 150

# Dry run — optimize but skip doc creation and sheet writes
python3 scripts/run_batch.py --limit 5 --dry-run
```

**Recommended rollout:** 50–80 rows per week to avoid unnatural crawl signals.

## What Each Script Does

### run_batch.py
The only script you need to call directly. On every run it automatically:
- **Refreshes the OAuth token** if expired (uses stored refresh token — no browser needed)
- **Rebuilds the sitemap cache** if older than 7 days (~60s, runs before first row)

Then for each "optimize" row:
1. Fetches blog post HTML → cleans to markdown
2. Calls `optimize_post.py` (optimization)
3. Calls `create_geo_doc.py` (Google Doc creation)
4. Writes Doc URL, `Done ✅`, and date back to the sheet
5. On failure: writes error to `Fail Reason`, sets `Failed ❌`, continues to next row

### optimize_post.py
Builds and sends the GEO optimization prompt with:
- **Consistency instruction** — locks output structure, varies prose phrasing
- **Format contract** — strict section order (H1 → sapo → body → table → FAQ → References)
- **Rotating expertise phrases** — 16-phrase pool across 4 groups, 2 picked randomly per post (~144 unique combinations across 528 posts)
- **Verified internal links** — pulled from local TF-IDF sitemap cache, zero tokens

### create_geo_doc.py
- Strips any annotation comments from the markdown
- Converts markdown → HTML (H1/H2/H3 → native heading styles, tables, lists, bold, links)
- Uploads HTML to Google Docs via Drive API (proper heading styles rendered natively)
- Post-processes: applies black 0.5pt borders to every table cell via Docs API
- Makes doc readable by anyone with the link

### find_internal_links.py
- Loads `wzorg_link_cache.json` (24,681 real URLs)
- Computes TF-IDF cosine similarity between post keywords and URL path tokens
- Returns top 5 most semantically relevant URLs — no API calls, instant

### build_sitemap_cache.py
- Recursively fetches all 11 worksheetzone.org sub-sitemaps
- Filters out hash-slug coloring pages, team member pages, utility pages
- Saves `wzorg_link_cache.json`
- Re-run when new content is published on the site

## GEO Optimization Rules (10 rules applied per post)

**Cardinal rule:** Preserve the author's voice. Make the minimum change needed. Never rewrite a sentence that already passes.

| Rule | What it does |
|------|-------------|
| 1 — Sapo | Rewrite opening paragraph: answer title in sentence 1, add "X is..." definition, name target audience, 40–60 words, no meta-commentary opener |
| 2 — Paragraph openers | Rewrite only the first sentence of each paragraph if it lacks a declarative statement, concrete detail, or independent meaning |
| 3 — Headings | Rewrite vague label headings (Benefits, Tips, Overview) to specific; add H3 for sections >300 words with no subheading |
| 4 — Citations | Embed verified source links for any statistic/research/standard already in post; append ## References |
| 5 — Definitions | Insert "A [term] is..." for each key term's first mention lacking a nearby definition (≤25 words) |
| 6 — Lists | Convert inline sentences with 3+ items to bulleted/numbered lists |
| 7 — Table | Add comparison table if post covers multiple categories and no table exists (max 3–5 columns) |
| 8 — FAQ | Append exactly 4 Q&A pairs (### Q: format, 40–80 word answers); audit existing FAQ if present |
| 9 — Expertise signals | Insert 2 first-person practitioner sentences ("we"/"in our experience") in separate sections |
| 10 — Internal links | Insert 3–4 verified worksheetzone.org links from local cache; hard cap 4; never invent URLs |

## Output Format (enforced by format contract)

Every optimized post follows this structure, every time:
```
# [Original H1 title — preserved exactly]
[Sapo — 40-60 words]
[Body sections — same H2/H3 count and hierarchy as original]
[Comparison table — only if applicable]
## Frequently Asked Questions
### Q: ...
[40-80 word answer]
### Q: ...  (×4 total)
## References  (only if Rule 4 added citations)
```

## Error Reference

| Error | Cause | Fix |
|-------|-------|-----|
| `oauth_token.pickle not found` | Auth not set up | Run the installer (see First-Time Setup) |
| `Token expired` | Refresh token itself revoked | Re-run the installer to re-authenticate |
| `Gemini error` | Gemini CLI not logged in | Run `gemini` and log in |
| `Claude error` | Claude CLI not authenticated | Run `claude` and log in |
| `Fetch failed` | Blog URL blocked or offline | Logged to Fail Reason; row skipped |
| `wzorg_link_cache.json not found` | Cache not built | Run `python3 scripts/build_sitemap_cache.py` |
| `Doc creator failed` | Docs/Drive API issue | Check Google Cloud Console — APIs must be enabled |

> **Note:** Expired OAuth tokens and stale link caches are handled automatically on every run. Manual intervention is only needed if the token file is missing or the refresh token itself was revoked.
